class amp.AMP(load=None, label=None, dblabel=None, extrapolate=True, fortran=True, **kwargs)[source]¶

Atomistic Machine-Learning Potential (AMP) ASE calculator

Parameters:
  • fingerprint (descriptor object) – Class representing local atomic environment. Can be only None and Behler for now. Input arguments for Behler are cutoff and Gs; for more information see docstring for the class Behler.
  • regression (descriptor object) – Class representing the regression method. Can be only NeuralNetwork for now. Input arguments for NeuralNetwork are hiddenlayers, activation, weights, and scalings; for more information see docstring for the class NeuralNetwork.
  • fingerprints_range (dict) –

    Range of fingerprints of each chemical species. Should be fed as a dictionary of chemical species and a list of minimum and maximun, e.g:

    >>> fingerprints_range={"Pd": [0.31, 0.59], "O":[0.56, 0.72]}
  • load (str) – Path for loading an existing parameters of AMP calculator.
  • label (str) – Default prefix/location used for all files.
  • dblabel (str) – Optional separate prefix/location for database files, including fingerprints, fingerprint derivatives, and neighborlists. This file location can be shared between calculator instances to avoid re-calculating redundant information. If not supplied, just uses the value from label.
  • extrapolate (bool) – If True, allows for extrapolation, if False, does not allow.
  • fortran (bool) – If True, will use fortran modules, if False, will not.
Raises:

RuntimeError

train(images, energy_goal=0.001, force_goal=0.005, overfitting_constraint=0.0, force_coefficient=None, cores=None, optimizer=<function fmin_bfgs at 0x04085A70>, read_fingerprints=True, overwrite=False)[source]¶

Fits a variable set to the data, by default using the “fmin_bfgs” optimizer. The optimizer takes as input a cost function to reduce and an initial guess of variables and returns an optimized variable set.

Parameters:
  • images (list or str) – List of ASE atoms objects with positions, symbols, energies, and forces in ASE format. This is the training set of data. This can also be the path to an ASE trajectory (.traj) or database (.db) file. Energies can be obtained from any reference, e.g. DFT calculations.
  • energy_goal (float) – Threshold energy per atom rmse at which simulation is converged.
  • force_goal (float) – Threshold force rmse at which simulation is converged. The default value is in unit of eV/Ang. If ‘force_goal = None’, forces will not be trained.
  • cores (int) – Number of cores to parallelize over. If not specified, attempts to determine from environment.
  • optimizer (object) – The optimization object. The default is to use scipy’s fmin_bfgs, but any optimizer that behaves in the same way will do.
  • read_fingerprints (bool) – Determines whether or not the code should read fingerprints already calculated and saved in the script directory.
  • overwrite (bool) – If a trained output file with the same name exists, overwrite it.

class amp.SaveNeighborLists(cutoff, hash_keys, images, label, log, train_forces, read_fingerprints)[source]¶

Neighborlists for all images with the given cutoff value are calculated and saved. As well as neighboring atomic indices, neighboring atomic offsets from the main cell are also saved. Offsets become important when dealing with periodic systems. Neighborlists are generally of two types: Type I which consists of atoms within the cutoff distance either in the main cell or in the adjacent cells, and Type II which consists of atoms in the main cell only and within the cutoff distance.

Parameters:
  • cutoff (float) – Cutoff radius, in Angstroms, around each atom.
  • hash_keys (list) – Unique keys, one key per image.
  • images (list) – List of ASE atoms objects (the training set).
  • label (str) – Prefix name used for all files.
  • log (Logger object) – Write function at which to log data. Note this must be a callable function.
  • train_forces (bool) – Determining whether forces are also trained or not.
  • read_fingerprints (bool) – Determines whether or not the code should read fingerprints already calculated and saved in the script directory.

  • train_forces (bool) – Determines whether or not forces should be trained.
  • _mp (object) – Multiprocess object.
  • overfitting_constraint (float) – The constant to constraint overfitting.
  • force_coefficient (float) – Multiplier of force RMSE in constructing the cost function. This controls how tight force-fit is as compared to energy fit. It also depends on force and energy units. Working with eV and Angstrom, 0.04 seems to be a reasonable value.
